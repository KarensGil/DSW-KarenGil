Propósito del sitio

Este sitio tiene como proposito definir y ver los aspectos mas relevantes de las hojas de estilo CSS y frameworks CSS, se espera capturar la mayor cantidad de información al respecto para que quede lo mas claro posible

Hojas de estilo en cascada (css)

Definición: Para Durango, A. (2015), Es el formato recomendado para las paginas escritas en formato HTML, la idea que se encuentra detras del desarrollo de CSS es separar la estructura de un doumento de su presentación o aspecto, segùn Cabello, J. C. O. (2006), Ofrecen un modo flexible para agregar estilos a un sitio web Este conjunto de reglas de formato rige el aspecto de las páginas y permite definir las fuentes, los colores, el formato y otras características de presentación.

¿Para qué sirven las hojas de estilo?

"Constituyen el complemento ideal para HTML o XHTML, su misión es definir la apariencia y el estilo de sus elementos". Cabello, J. C. O. (2006). Diseño de páginas Web con XHTML, JavaScript y CSS. Grupo Editorial RA-MA.

Pilamunga Poveda, E. M., & DT-López Sevilla, G. M. (2012), define algunas de las cosas que se pueden hacer usando este lenguaje, dentro de las cuales podemos encontrar:
*Permite definir toda la pagina Web de una sola vez
*Aplicar estilos visibles en un trozo de la pagina
*definir incluso varios estilos diferentes para una sola etiqueta
*permite aplicar formato de una forma más exacta
Una vez aparece la sintaxis CSS se vuelve más fácil maquetar las paginas, ya que las herramientas dadas son mucho más prácticas.

Frameworks CSS:

“Son herramientas usadas que permiten encapsular las tareas repetitivas en módulos genéricos fácilmente reutilizables, permiten al diseñador centrarse en los elementos únicos de cada diseño en los que puede aportar valor” (Uniwebsidad, Capitulo 5: Frameworks).
Tres de los mejores frameworks son:

1. Bootstrap:

Se le atribuye el mérito de incluir el diseño receptivo a gran escala, fue el primero en promover la filosofía de “dispositivos móviles primero”
Lo bueno:
-Permite la creación rápida de prototipos: Sin necesidad de preocuparse por posicionamiento CSS o compatibilidad con el navegador, con escribir en HTML y aplicar las clases de CSS apropiadas hace que la capacidad de respuesta cobre vida
-Gran ecosistema: Tiene el ecosistema mas grande de marcos de front-end La cantidad de diseños de sitios web, temas, paneles de administración, componentes de la interfaz de usuario, etc., creados con Bootstrap es alucinante y sigue mejorando
-Una gran colección de componentes: Bootstrap ofrece, listos para usar, casi todos los componentes de la interfaz de usuario que probablemente necesite. Navegación, formularios, tarjetas, modales, botones, insignias, barras de progreso, alertas, etc

2. Foundation:

El marco de front-end más receptivo del mundo, es intuitivo y fácil de memorizar
Lo bueno:
-Herramientas completas: Tiene como objetivo solucionar casi todo tipo de problemas de front-end
-Flexibilidad extrema: El diseñador tiene control total de sus interfases, no impone ningún estilo.
- Más que solo componentes de la interfaz de usuario: Los desarrolladores han incluido un sistema avanzado de imágenes receptivas, un componente de tabla de precios, validación de formularios, soporte de derecha a izquierda, incrustaciones receptivas y más

3. Bulma:

Es relativamente nuevo en el mundo de los marcos CSS, pero se ha posicionado rápidamente; este no tiene componentes de JavaScript, se ha basado estrictamente en la sintaxis CSS y valores predeterminados elegantes
Lo bueno
-Muy popular
-Clases extremadamente legibles: Tiene un sistema simple y poderoso para crear cuadriculas, llamadas mosaico
-Elegante
